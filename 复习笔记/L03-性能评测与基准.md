## 二、核心知识点

### 2.1 基准程序(Benchmark)

- **定义**:性能测试程序,假设能刻画某一类应用问题的处理和数据移动特征。
- **基准程序组(benchmark suite)**:一套基准程序 + 控制测试条件及过程的特定规则(输入数据、输出结果、性能指标)。
- **分类**:
  - 按应用类型:科学计算、人工智能、商业应用、网络服务、多媒体应用、信号处理等。
  - 按复杂程度:**微基准程序**(测系统某一方面,如 CPU/存储器/IO 速度)与**宏基准程序**(测系统总体性能)。

代表性基准程序组:

| 类型 | 名称 | 测量内容 |
|---|---|---|
| 微基准程序 | **Linpack、HPCG** | 数值计算(稠密 / 稀疏) |
| 微基准程序 | **Graph500** | 非数值计算(图计算) |
| 微基准程序 | **AIPerf** | 人工智能 |
| 微基准程序 | **Lmbench** | 系统调用和数据移动 |
| 微基准程序 | **Stream** | 存储器带宽 |
| 微基准程序 | **OSU** | 大规模网络消息传递 |
| 宏基准程序 | **NAS NPB** | 并行计算(CFD) |
| 宏基准程序 | **ECP Proxy Apps Suite** | 超算代理应用 |
| 宏基准程序 | **SPEC** | CPU 混合基准程序系列 |
| 宏基准程序 | **TPC** | 事务处理和数据库商业应用 |

### 2.2 Linpack(重点)

- 采用 **LU 分解的高斯消元法**求解大型稠密线性方程组 $Ax=b$,由美国田奴大学 Jack Dongarra 创作维护。已成为评测高性能计算机性能的事实标准(TOP500 依据)。
- **关键指标**:
  - **Rpeak**:被测系统的**理论峰值速度**。
  - **Rmax**:可达到的**持续最大速度**。
  - **Nmax**:达到 Rmax 时的求解问题规模(矩阵阶数)。
- **浮点运算次数公式**:$\dfrac{2}{3}N^{3}-2N^{2}$
- **Rmax 公式**:$R_{\max}=\dfrac{\dfrac{2}{3}N^{3}-2N^{2}}{\text{运行时间}}$

> 例:TOP500(2023.06)第 1 名 Frontier,Rmax=1194.00 PFlop/s,Rpeak=1679.82 PFlop/s,功耗 22,703 kW,共 8,699,904 核。

### 2.3 HPCG

- 共轭梯度法求解大型系数矩阵方程组 $Ax=b$,由 Sandia 国家实验室 Michael A. Heroux 维护。
- 源自非定常数非线性偏微分方程组,迭代求解频繁存取不规则数据。
- **对系统要求**:高带宽、低延时、高 CPU 主频(计算 / 访存比低)。
- Linpack 计算密集(计算/访存比高),HPCG 访存密集(计算/访存比低)。

### 2.4 Graph500

- 在庞大无向图中采用**宽度优先搜索(BFS)**。三个计算核心:① 生成图并以 CSR/CSC 压缩存储;② 并行 BFS 检索;③ 单源最短路径。
- 6 种数据规模:Toy($2^{26}$ 顶点,~17GB)、Mini($2^{29}$,~137GB)、Small($2^{32}$,~1.1TB)、Medium($2^{36}$,~17.6TB)、Large($2^{39}$,~140TB)、Huge($2^{42}$,~1.1PB)。
- 指标:**GTEps**(Giga Traversed Edges per second) = $m / \text{time}_{k2}(n)$,$m$ 为遍历的边数目。

### 2.5 AIPerf

- 清华陈文光团队(鹏城实验室、清华、中科院计算所)提出。以 **AutoML** 作为代表性自适应工作负载,为 AI-HPC 建立端到端基准测试套件。
- 用每秒运算 **OPS** 作为基准分数。

### 2.6 Lmbench & Stream

- **Lmbench**(SGI, Larry McVoy):测量操作系统开销、处理器、高速缓存、存储器、网络、磁盘之间数据传输能力。
- **Stream**(SGI, John McCalpin):测量持续存储器带宽(MB/s)及相应计算速率。

Stream 内核及计算量:

| 名称 | 代码 | 字节 / 迭代 | 浮点数 / 迭代 |
|---|---|---|---|
| Copy | `a(i)=b(i)` | 16 | 0 |
| Scale | `A(i)=q*b(i)` | 16 | 1 |
| Sum | `A(i)=b(i)+c(i)` | 24 | 1 |
| Triad | `A(i)=b(i)+q*c(i)` | 24 | 2 |

- **机器平衡值** = 峰值浮点速度(flop/s) / Triad 操作的持续存储器带宽(word/s)。

### 2.7 性能与成本指标

- **执行时间**:在目标机器上运行应用程序并用墙上时钟测量所花时间。
- **处理速度**:如 Gflop/s,需要保证的速度。
- **系统吞吐率**:多个作业同时执行时,单位时间内所处理的作业数。
- **利用率** = 给定计算机可达到速度 / 峰值速度。低利用率总意味着差的程序或编译器。
- **成本有效性** = 速度 / 总成本(如 GF/$)。

### 2.8 三种工作负载测量指标

| 工作负载类型 | 工作负载单位 | 速度单位 |
|---|---|---|
| 执行时间 | 秒、CPU 时钟 | 应用 / 每秒 |
| 指令数(动态指令数) | 百万 / 十亿条指令 | MIPS 或 BIPS |
| 浮点操作数 | flop、Mflop、Gflop | Mflop/s、Gflop/s |

- 执行时间依赖计算机系统;指令数依赖指令集体系结构;浮点操作数通常与体系结构无关(最稳定)。

**浮点数(flop)计数规则**:

| 操作 | Flop | 规则 |
|---|---|---|
| `A[2*i]=B[j-1]+1.5*C-2` | 3 | 加 / 减 / 乘每个 1 flop,下标不计,赋值不分开计 |
| `X=Y` | 1 | 孤立赋值 1 flop |
| `if (X>Y) Max=2.0*X` | 2 | 比较 1 flop |
| `X=(float)i+3.0` | 2 | 类型转换 1 flop |
| `X=Y/3.0+sqrt(Z)` | 9 | 除或开方计 4 flop |
| `X=sin(Y)-exp(Z)` | 17 | 正弦、指数等计 8 flop |

### 2.9 并行计算机性能模型

- **执行时间分解**(无重叠):$T=T_{\text{comp}}+T_{\text{par}}+T_{\text{interact}}$(计算执行 + 并行性操作 + 交互操作)。
  - 并行性操作:进程管理、分组操作、进程查询。
  - 交互操作:同步(barrier、lock、critical)、通信(点到点、集合)。

**BSP 模型(Bulk Synchronous Parallelism, Valiant)**:
- 由处理器-存储器对集合 + 点对点通信网络 + 全处理器 barrier 同步机制组成。
- 计算由一系列 **superstep** 组成;每个 superstep 内进程执行本地计算并发通信请求;superstep 末尾做 barrier 同步,所有通信完成。
- 参数:$p$=处理器数,$l$=barrier 延迟,$g$=每字通信代价,$s$=处理器速度($l,g,s$ 均以 FLOPS 度量)。
- 单 superstep 中每处理器最多发/收 $h$ 条消息(h-relation)。
- **superstep 时间** = 任一处理器本地操作数最大值 $+ g\cdot h + l$。

**LogP 模型(Culler, Berkeley)**:
- $P$:处理器数;$L$:消息传输延迟;$o$:发送 / 接收开销;$g$:连续 send/recv 间隔($1/\text{BW}$)。
- 容量受限(到 / 离任一处理器最多 $L/g$ 条消息)。
- 消息从 A 到 B 总时间:$L + 2o$。

**Hockney 通信模型**(后续统一使用):
- 参数:$t_s$(启动延迟), $t_h$(每跳时间), $t_b$(每字节时间)。
- 点到点通信(消息长度 $m$ 字节):
$$t_{\text{comm}}(m)=t_s + m\cdot t_b \;=\; \alpha + \frac{m}{\beta}$$
  其中 $\alpha=t_s$ 为网络延迟,$\beta=R=1/t_b$ 为网络带宽(MB/s)。
- 集合通信(消息长度 $m$, $n$ 个处理器):
$$T(m,n)=t_0(n)+\frac{m}{r_\infty(n)}$$
  $t_0$ 为启动时间,$r_\infty$ 为渐近带宽(MB/s)。

**开销测量**:
- 乒乓方案(测点对点时延):循环多次取 max/min/avg;定时器仅在 1 个处理器上执行,不受时间同步影响。
- 集合通信测量:通过广播启动所有结点、规约停止所有结点,只在结点 0 计时来回避时间同步问题。

### 2.10 OSU 基准测试程序

MPI microbenchmark(MPI v4.1.5),包含:
- Host-based:Point-to-Point(osu_latency / osu_bw / osu_bibw / osu_mbw_mr / osu_multi_lat 等)、Blocking Collective(osu_allreduce / osu_allgather / osu_bcast 等)、Non-Blocking Collective(osu_iallreduce 等)、One-sided(osu_put/get_bw、osu_acc_latency、osu_cas_latency 等)、Startup(osu_init / osu_hello)。

## 三、重点公式 / 定律

### 3.1 基于阶段并行模型(Phase-Parallel Model)

设串行程序 $C$ 由 $k$ 个分计算阶段 $C_1,\dots,C_k$ 组成,阶段 $C_i$ 的并行度 $DOP_i$、工作负载 $W_i$(Mflop)、单处理器执行时间 $T_1(i)$。

- **n 个处理器上阶段 $C_i$ 的并行执行时间**(当 $1\le n\le DOP_i$):$T_n^{(i)}=T_1(i)/n$
- **n 个处理器并行总时间**:
$$T_n=\sum_{1\le i\le k}\frac{T_1(i)}{\min(DOP_i,\,n)}+T_{\text{par}}+T_{\text{interact}}$$
- **关键路径**(无限结点、不计开销):
$$T_\infty=\sum_{1\le i\le k}\frac{T_1(i)}{DOP_i}$$
- **最大并行性** $N_{\max}$:使 $T_n=T_\infty$ 的最小 $n$ 值,$N_{\max}=\max_i DOP_i$。
- **界限**(不计开销):$\max\!\left(\dfrac{T_1}{n},\,T_\infty\right)\le T_n\le \dfrac{T_1}{n}+T_\infty$。

性能指标汇总表(必背):

| 指标记号 | 术语 | 定义 |
|---|---|---|
| $T_1$ | 串行时间 | $T_1=\sum_{1\le i\le k}T_1(i)$ |
| $T_n$ | n 结点并行时间 | $\sum_i T_1(i)/\min(DOP_i,n)+T_{\text{par}}+T_{\text{interact}}$ |
| $T_\infty$ | 关键路径 | $\sum_i T_1(i)/DOP_i$ |
| $P_n$ | n 结点速度 | $P_n=W/T_n$ |
| $S_n$ | n 结点加速比 | $S_n=T_1/T_n$ |
| $E_n$ | n 结点效率 | $E_n=S_n/n=T_1/(nT_n)$ |
| $U_n$ | n 结点利用率 | $U_n=P_n/(nP_{\text{peak}})$ |
| $T_o$ | 总开销 | $T_o=T_{\text{par}}+T_{\text{interact}}$ |
| — | 平均并行性 | $T_1/T_\infty$ |
| — | 平均开销 | $T_o/W$ |
| — | 平均颗粒度 | $W/T_o$ |

### 3.2 Amdahl 定律(固定问题规模 / Fixed-load)

设固定工作负载 $W=\alpha W+(1-\alpha)W$,$\alpha$ 为 $W$ 中**必须串行执行的百分比**。忽略其他开销,固定负载加速比:

$$S_n=\frac{W}{\alpha W+\dfrac{(1-\alpha)W}{n}}=\frac{n}{1+(n-1)\alpha}\;\xrightarrow{n\to\infty}\;\frac{1}{\alpha}$$

**含义**:
- 最大加速比上限为 $1/\alpha$,串行成分是瓶颈,$\alpha$ 增大加速比成比例减少。
- 获取好加速比的关键是使串行瓶颈 $\alpha$ 尽可能小。
- 应设法使较大(可并行)部分执行更快。

**扩展 Amdahl 定律**(考虑总开销 $T_o$):
$$S_n=\frac{W}{\alpha W+\dfrac{(1-\alpha)W}{n}+T_o}=\frac{n}{1+(n-1)\alpha+\dfrac{nT_o}{W}}\;\xrightarrow{n\to\infty}\;\frac{1}{\alpha+\dfrac{T_o}{W}}$$

含义:并行程序性能不仅受串行瓶颈限制,也受平均开销 $T_o/W$ 限制;应同时减小串行瓶颈、增大平均颗粒度。

### 3.3 Gustafson 定律(固定时间 / Fixed-time)

Amdahl 假设问题规模固定,导致"用大系统解小问题"的报酬递减。Gustafson 提出**固定时间**:机器规模增大时同步扩大问题规模。

**推导**(Gustafson–Barsis 形式):设并行执行中串行部分时间为 $\tau(n)$、并行部分时间为 $\pi(n)$,总时间 $T(n)=\tau(n)+\pi(n)$。令串行比例 $g=\dfrac{\tau(n)}{\tau(n)+\pi(n)}$,则"等价串行时间"为 $\tau(n)+n\pi(n)$,固定时间加速比:

$$S_n=\frac{\tau(n)+n\pi(n)}{\tau(n)+\pi(n)}=g+ n(1-g)=(1-g)n+g$$

即 **$S_n = n - (n-1)g$**(其中 $g$ 为并行执行中串行部分占比)。

**固定时间加速比**(中文讲义形式):工作负载扩大为 $W'=\alpha W+(1-\alpha)nW$,并行执行时间仍为 $W$,而其等价串行时间也为 $W'=\alpha W+(1-\alpha)nW$,故

$$S'_n=\frac{\text{扩大负载的串行时间}}{\text{扩大负载的并行时间}}=\frac{\alpha W+(1-\alpha)nW}{W}=\alpha+(1-\alpha)n$$

**含义**:若工作负载可扩展以维持固定执行时间,固定时间加速比是 $n$ 的线性函数;问题规模扩张到匹配扩展的计算能力时,串行部分不再是瓶颈。关键是只让可并行部分扩展 $n$ 倍,串行部分保持不变。

**扩展(含开销)**:
$$S'_n=\frac{\alpha W+(1-\alpha)nW}{W+T_o}=\frac{\alpha+(1-\alpha)n}{1+\dfrac{T_o}{W}}$$

### 3.4 Sun-Ni 定律(存储器受限)

限定存储器空间求解尽可能大的问题,是 Amdahl 与 Gustafson 的广义化。设 $M$ 为单结点存储器容量,$G(n)$ 为工作负载并行部分的扩展倍数(反映存储器容量增加 $n$ 倍时工作负载增加量)。

- **不考虑开销**:
$$S'_n=\frac{\alpha W+(1-\alpha)G(n)W}{\dfrac{\alpha W+(1-\alpha)G(n)W}{n}}=\frac{\alpha+(1-\alpha)G(n)}{\alpha+\dfrac{(1-\alpha)G(n)}{n}}$$

- **考虑开销**:
$$S'_n=\frac{\alpha+(1-\alpha)G(n)}{\alpha+\dfrac{(1-\alpha)G(n)}{n}+\dfrac{T_o}{W}}$$

**$G(n)$ 取值**:
- $G(n)=1$:问题规模固定,退化为 **Amdahl** 定律。
- $G(n)=n$:存储容量与工作负载均增加 $n$ 倍,退化为 **Gustafson** 定律。
- $G(n)>n$:计算负载增长快于存储,存储器受限加速比高于固定负载 / 固定时间加速比。

### 3.5 等效率函数(Iso-efficiency)

- **系统**:由应用(工作负载)和并行计算机平台构成的一对。
- **效率**:$E=f(W,n)$。令效率固定为常数 $E$,对工作负载 $W$ 求解,得到的函数称为**等效率函数**。
- **推导**:由效率 $E=\dfrac{T_1}{T_1+T_o}=\dfrac{1}{1+T_o/W}$,令 $E$ 为常数,解出
$$E(T_1+T_o)=T_1\;\Rightarrow\;E\cdot T_o=T_1(1-E)\;\Rightarrow\;T_1=\frac{E}{1-E}\,T_o$$
定义 $K=\dfrac{E}{1-E}$,则保持效率 $E$ 所需工作负载($W=T_1$):
$$W = K\cdot T_o = \frac{E}{1-E}\,T_o$$
- **含义**:等效率值越小,机器规模增大时为保持相同效率所需增加的工作负载越小,**可扩展性越好**。

效率、加速比、开销关系(来自 P100–101 推导):
$$T_p = \frac{T_1}{p}+\frac{T_0}{p}\cdot\frac{1}{?},\quad S_p=\frac{T_1}{T_p},\quad E_p=\frac{S_p}{p}=\frac{T_1}{p\cdot T_p}=\frac{1}{1+\dfrac{T_0}{T_1}}$$
(令 $E=$ const,得 $T_1=\dfrac{E}{1-E}\,T_0$,即等效率 $W=T_1=K\cdot T_0$,$K=\dfrac{E}{1-E}$。)

> 三种等性能模型:**等效率(isoefficiency)**、**等速度(isospeed)**、**等利用率(isoutilization)**。

### 例 1:吞吐率 / 利用率 / 成本有效性(讲义 P15–17)

**假设一个 256 结点的并行系统,三个程序 P1/P2/P3 的性能如下:**

| 程序 | 执行时间(s) | 速度(Gflop/s) | 加速比 | 利用率 |
|---|---|---|---|---|
| P1 | 0.16 | 9 | 90 | 13% |
| P2 | 0.56 | 23 | 233 | 34% |
| P3 | 1.40 | 3.8 | 86 | 6% |

- **执行时间**:要求程序 0.5s 完成,只有 P1 能满足。
- **处理速度**:要求 10 Gflop/s,只有 P2 能满足。
- **吞吐率**(以 P1 为对象):每 0.16s 一个 P1,即每秒 $1/0.16=6.25$ 个 P1。

**提高吞吐率的两种方式**(以 P1 为例):
P1 工作负载 $=9\times 0.16=1.44$ Gflop。

- **流水线**:两级流水,每级 128 个结点,第一级 0.14s、第二级 0.11s。总执行时间 $0.14+0.11=0.25$s(从 0.16s 增加),但吞吐率 $\to 1/0.14=7.14$ 个/s $\approx 10.34$ Gflop/s。
- **并行**:一个结点上 P1 执行时间 14s,256 个结点各跑一个 P1,吞吐率 $=256/14=18.29$ 个/s $\approx 25.6$ Gflop/s。

**利用率与成本有效性**(P17,假设每节点峰值 266 Mflop/s,256 结点峰值 $=266\times 256=68128$ Mflop/s $\approx 68$ Gflop/s;每 CPU 小时收费 \$10):

| 指标 | 1 个结点 | 256 个结点 |
|---|---|---|
| 速度 | 100 Mflop/s | 9 Gflop/s |
| 时间 | 14.4 s | 0.16 s |
| 利用率 | $100/266=37.6\%$ | $9/68=13.3\%$ |
| 总成本 | $10\times 14.4/3600=\$0.04$ | $10\times 256\times 0.16/3600=\$0.11$ |
| **成本有效性** | $100/0.04=2.5$ GF/\$ | $9/0.11=81$ GF/\$ |

> 注意:此处"1 个结点"行速度 100 Mflop/s、时间 14.4s 对应同一工作负载在单结点的执行,与"256 结点"行的 9 Gflop/s、0.16s 对应。

### 例 2:阶段并行算法性能指标(讲义 P81–82,★核心计算题)

**题意**:给定如图所示的粗粒度阶段并行算法,参数 $n=256$,并行性开销忽略不计。四个阶段:

| 阶段 | 名称 | $DOP$ | 工作负载 | 单结点时间 | 消息长度 |
|---|---|---|---|---|---|
| DP | 全交换(alltoall) | 8192 | 84 Mflop | 4.12 s | $16.7/n^2$ MB |
| HT | 融合目标报告的规划 | 1 | 2.88 Mflop | 0.04 s | 80 KB |
| BF | 广播 | 256 | 1314 Mflop | 9.64 s | 100 flop/报告 |
| TD | (后续阶段) | 256 | 46 Mflop | 0.57 s | — |

**(1) 最大并行性**:
$$N_{\max}=\max\{8192,\,1,\,256,\,256\}=8192$$

**(2) 总工作负载**:
$$W=84+2.88+1314+46=1447\ \text{Mflop}$$

**(3) 串行执行时间**:$T_1=14.37$ s(即 $4.12+0.04+9.64+0.57$)。

**(4) 关键路径**(无限结点,不计开销):
$$T_\infty=\frac{4.12}{8192}+\frac{0.04}{1}+\frac{9.64}{256}+\frac{0.57}{256}=0.08\ \text{s}$$

**(5) 最大性能值**:
$$P_\infty=\frac{W}{T_\infty}=\frac{1447}{0.08}=18087\ \text{Mflop/s}\approx 18\ \text{Gflop/s}$$

**(6) 平均并行性**:
$$\frac{T_1}{T_\infty}=\frac{14.37}{0.08}\approx 180$$

**(7) 交互开销**:
$$T_{\text{interact}}=T_{\text{alltoall}}+T_{\text{bcast}}+T_{\text{reduce}}$$

**(8) 总执行时间**:
$$T_n=T_{\text{comp}}+T_{\text{interact}}=\frac{14.33}{n}+T_{\text{interact}}$$
(其中计算部分 $T_{\text{comp}}=14.33/n$,因为除 HT 串行阶段外其余阶段在 $n=256$ 时已充分并行:$4.12/8192+0.04+9.64/256+0.57/256\approx 0.08+0.04\approx$ 注:此处讲义取 $14.33/n$ 形式表达。)

### 例 3:Amdahl 加速比上限(讲义 P86,★核心计算题)

**题意**:考察例 1(阶段并行算法),在以下条件下估计 Amdahl 加速比上限:
(1) 不计所有开销;
(2) 考虑所有开销且 $T_{o,\infty}\approx T_{o,256}=0.0479$ s。

**已知**:工作负载 $W=14.37$ s(以串行时间表示),串行部分是 HT 阶段($T=0.04$ s)。

**(1) 串行比例**:
$$\alpha=\frac{0.04}{14.37}=0.00278=0.278\%$$

**不计开销的 Amdahl 加速比上限**:
$$S_n\to\frac{1}{\alpha}=\frac{1}{0.00278}=359$$

**(2) 平均开销**:
$$\frac{T_{o,\infty}}{W}=\frac{0.0479}{14.37}=0.00333$$

**扩展 Amdahl 定律的较严格上限**:
$$S_n\to\frac{1}{\alpha+\dfrac{T_o}{W}}=\frac{1}{0.00278+0.00333}=163$$

### 例 4:固定时间加速比(讲义 P94,★核心计算题)

**题意**:考察例 1,设问题参数 $N=256$,工作负载近似为:
- flop 形式:$W=0.011N^2+2.8N+2.88$ Mflop
- 时间形式:$0.00016N^2+0.015N+0.04$ s

$N=256$ 时 $W=1447$ Mflop(或 14.37 s)。串行 HT 占比 $\alpha=\dfrac{0.04}{14.37}=0.00278=0.278\%$。求 $n=128$ 结点时的固定时间加速比。

**代入固定时间加速比公式**:
$$S'_n=\alpha+(1-\alpha)n=0.00278+0.99722\times 128=127.65$$

**扩大后的工作负载(等价串行时间)**:
$$W'=\alpha W+(1-\alpha)nW=127.65\times 14.37=1834\ \text{s}$$

**问题规模扩展讨论**:
- 代入工作负载时间表达式 $W'=1834$ s 求解,得 $N=3339$(扩大约 13 倍)。
- 进一步代入 flop 表达式,得 $W'=131996$ Mflop(扩大约 91 倍)。

### 例 5:含开销的扩展加速比(讲义 P95,★核心计算题)

**题意**:在 $n=128$ 结点上,假设 $T_o=T_{\text{alltoall}}+T_{\text{bcast}}+T_{\text{reduce}}=2.75$ s,求扩展加速比。

**代入**:
$$S'_n=\frac{\alpha+(1-\alpha)n}{1+\dfrac{T_o}{W}}=\frac{0.00278+0.99722\times 128}{1+\dfrac{2.75}{14.37}}=\frac{127.65}{1+0.1909}=107$$

### 例 6:计算常数 π 的等效率函数(讲义 P102–105,★核心计算题)

**题意**:用矩形法数值积分计算 $\displaystyle\pi=\int_0^1\frac{4}{1+x^2}\,dx$,采用循环分配(cyclic scheme)将计算分配到各处理器,各处理器部分和再求和(归约)。

**已知**:
- $n$:区间 $[0,1]$ 的分段数(问题规模)。
- 算法计算复杂度(串行时间 / 串行工作负载):$T_1=W=6n$。
- 每个处理器的网格节点数:$m=\lceil n/p\rceil\le n/p+1$($p$ 为处理器数)。
- 每个处理器的计算复杂度:$W_p=6m=6\lceil n/p\rceil\le 6(n/p+1)$。

**并行执行时间**(计算 + 集合归约,采用 $\log_2 p$ 步树形归约):
$$T_p=\frac{6n}{p}+6+\log_2 p$$

**加速比**:
$$S_p=\frac{T_1}{T_p}=\frac{6n}{\dfrac{6n}{p}+6+\log_2 p}$$

**效率**:
$$E_p=\frac{S_p}{p}=\frac{6n}{6n+6p+p\log_2 p}$$

**等效率函数推导**:令 $E=$ const,由 $E=\dfrac{1}{1+T_o/T_1}$ 得
$$T_1=K\cdot T_o,\quad K=\frac{E}{1-E}$$
此处 $T_o=6p+p\log_2 p$(总开销),$T_1=W=6n$,故
$$6n=K(6p+p\log_2 p)\ \Rightarrow\ n=K\!\left(p+\frac{p}{6}\log_2 p\right)$$

**结果**(讲义取值示例):
- $E=0.5$ 时:$K=1$,$n=p+\dfrac{p}{6}\log_2 p$,当 $p=8$ 时 $n=12$;当 $p=64$ 时 $n=128$。
- 当 $p$ 较大时,等效率 $\Theta(p\log_2 p)$,即问题规模需以 $p\log p$ 量级增长才能维持固定效率。

### 例 7:有限差分法的等效率函数(讲义 P106–107)

**题意**:$n\times n$ 网格上做有限差分,迭代 $M$ 次。每个处理器处理 $\dfrac{n}{p}\times\dfrac{n}{p}$ 个网格节点的子矩形区域,每次迭代后同步。

**已知**:
- 串行时间 / 工作负载:$T_1=W=6n^2 M$($M$ 为迭代次数)。
- 并行时间:$T_p=\dfrac{6n^2 M}{p}+M\log_2 p$。

**加速比**:
$$S_p=\frac{T_1}{T_p}=\frac{6n^2}{\dfrac{6n^2}{p}+\log_2 p}$$
(消去 $M$)

**等效率函数**:总开销 $T_o=M\cdot p\log_2 p$。令 $E=$ const:
$$W=K\cdot T_o\ \Rightarrow\ 6n^2 M=K\cdot M p\log_2 p\ \Rightarrow\ 6n^2=K\cdot p\log_2 p$$
故等效率 $W=6n^2\propto \Theta(p\log_2 p)$。

### 例 8:两个矩阵乘方案的可扩展性比较(讲义 P108–109,★核心计算题)

**题意**:给定两个 $N\times N$ 矩阵相乘方案 A 和 B,相同处理器数 $n$。串行时间 $T_1=cN^3$,效率 $E=T_1/(nT_n)$。执行时间表达式第一项为计算时间,第二项为通信开销。系统 B 的开销仅为系统 A 的一半,但代价是计算时间增加一倍。

| 方案 | 并行执行时间 $T_n$ | 效率 $E$ |
|---|---|---|
| A | $\dfrac{cN^3}{n}+\dfrac{bN^2}{n}$ | $E_A=\dfrac{1}{1+\dfrac{bn}{cN}}$ |
| B | $\dfrac{2cN^3}{n}+\dfrac{bN^2}{2n}$ | $E_B=\dfrac{1}{2+\dfrac{bn}{2cN}}$ |

**问题**:要保持 (1) $E=1/3$ 和 (2) $E=1/4$,哪个系统可扩展性更好?工作负载 $W=N^3$。

**(1) $E=1/3$**:
- 系统 A:$\dfrac{1}{3}=\dfrac{1}{1+\dfrac{bn}{cN}}\Rightarrow \dfrac{bn}{cN}=2\Rightarrow N=\dfrac{bn}{2c}$,故
$$W_A=\left(\frac{b}{2c}\right)^3 n^{1.5}$$
- 系统 B:类似解得 $W_B=\left(\dfrac{b}{2c}\right)^3 n^{1.5}$。

**结论(1)**:两者等效率相等,$W\propto n^{1.5}$,**可扩展能力相同**。

**(2) $E=1/4$**:
- 系统 A:$\dfrac{1}{4}=\dfrac{1}{1+\dfrac{bn}{cN}}\Rightarrow \dfrac{bn}{cN}=3\Rightarrow N=\dfrac{bn}{3c}$,故
$$W_A=\left(\frac{b}{3c}\right)^3 n^{1.5}$$
- 系统 B:$W_B=\left(\dfrac{b}{4c}\right)^3 n^{1.5}$。

**结论(2)**:$W_B<W_A$,系统 B **等效率更小,可扩展性更好**。

## 五、考点与易错点

1. **Linpack 浮点次数公式**务必记牢:$\dfrac{2}{3}N^3-2N^2$;$R_{\max}=\dfrac{\frac{2}{3}N^3-2N^2}{\text{运行时间}}$。注意 $N$ 是矩阵阶数(=问题规模 $N_{\max}$)。
2. **Rpeak / Rmax / Nmax** 区分:Rpeak 理论峰值、Rmax 实测持续最大、Nmax 达到 Rmax 时的问题规模。
3. **flop 计数规则**:加减乘 = 1 flop、除 / 开方 = 4 flop、sin/exp 等 = 8 flop、类型转换 = 1 flop、比较 = 1 flop、下标不计、赋值不分开计。
4. **加速比 / 效率 / 利用率定义不要混淆**:
   - 加速比 $S_n=T_1/T_n$(基准是串行时间)。
   - 效率 $E_n=S_n/n=T_1/(nT_n)$。
   - 利用率 $U_n=P_n/(nP_{\text{peak}})$(基准是峰值速度)。
5. **Amdahl**:$\alpha$ 是**串行比例**,$S_n=\dfrac{n}{1+(n-1)\alpha}$,上限 $1/\alpha$。问题规模**固定**。
6. **Gustafson**:$g$ 是并行执行中串行部分占比,$S_n=n-(n-1)g$(或 $\alpha+(1-\alpha)n$),问题规模**随系统扩大**。注意 Gustafson 的串行比例定义与 Amdahl 不同(Gustafson 在并行执行时间里度量)。
7. **Sun-Ni** 通过 $G(n)$ 统一两者:$G(n)=1$ 退化为 Amdahl,$G(n)=n$ 退化为 Gustafson。
8. **等效率函数**:等效率值越小,可扩展性越好;推导核心是令 $E=$ const,由 $E=\dfrac{1}{1+T_o/W}$ 得 $W=K\cdot T_o$,$K=\dfrac{E}{1-E}$。
9. **关键路径** $T_\infty$ 与 **最大并行性** $N_{\max}=\max_i DOP_i$ 的概念易混;关键路径是"无限结点不计开销"的时间,最大并行性是使 $T_n=T_\infty$ 的最小 $n$。
10. **Hockney 模型**:$t_{\text{comm}}=\alpha+m/\beta$,$\alpha$ 是延迟(启动时间),$\beta$ 是带宽;集合通信 $T(m,n)=t_0(n)+m/r_\infty(n)$。
11. **BSP** superstep 时间 = 本地操作最大值 $+gh+l$;**LogP** 点对点总时间 $=L+2o$。
12. **乒乓测量**回避时间同步问题(定时器只在 1 个处理器上);集合通信用"广播启动 + 规约停止 + 仅结点 0 计时"。
