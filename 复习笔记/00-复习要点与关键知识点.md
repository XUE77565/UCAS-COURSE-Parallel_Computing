## 一、必背公式速查(★计算题核心,源自 L03)

### 1.1 加速比 / 效率 / 利用率(定义,勿混)

| 量 | 公式 | 含义 |
|---|---|---|
| 加速比 | $S_n = T_1/T_n$ | 基准是**串行时间** |
| 并行效率 | $E_n = S_n/n = T_1/(nT_n)$ | 每个处理器的"性价比" |
| 利用率 | $U_n = P_n/(nP_{\text{peak}})$ | 基准是**峰值速度** |
| 总开销 | $T_o = T_{\text{par}} + T_{\text{interact}}$ | 并行性开销 + 交互开销 |

### 1.2 三大可扩展性定律

**Amdahl 定律(固定问题规模 / 强可扩展)**
$$S_n = \frac{n}{1+(n-1)\alpha},\quad \lim_{n\to\infty}S_n=\frac{1}{\alpha}$$
- $\alpha$ = 工作负载中**必须串行**的比例。
- 上限 $1/\alpha$:串行成分是瓶颈。问题规模**固定**。
- 扩展(含开销):$S_n=\dfrac{n}{1+(n-1)\alpha+nT_o/W}\to\dfrac{1}{\alpha+T_o/W}$

**Gustafson 定律(固定时间 / 弱可扩展)**
$$S_n = n-(n-1)g = \alpha+(1-\alpha)n$$
- $g$(或 $\alpha$)= **并行执行时间中**串行部分占比(注意:与 Amdahl 的 $\alpha$ 度量时机不同)。
- 若问题规模随系统扩大,加速比是 $n$ 的**线性**函数,不再受串行瓶颈限制。

**Sun-Ni 定律(存储器受限,统一形式)**
$$S'_n=\frac{\alpha+(1-\alpha)G(n)}{\alpha+\dfrac{(1-\alpha)G(n)}{n}+\dfrac{T_o}{W}}$$
- $G(n)=1$ → 退化为 **Amdahl**;$G(n)=n$ → 退化为 **Gustafson**;$G(n)>n$ → 加速比更高。

### 1.3 等效率函数(Iso-efficiency)★

由 $E=\dfrac{T_1}{T_1+T_o}=\dfrac{1}{1+T_o/W}$,令效率 $E$ 为常数,得保持效率所需工作负载:

$$\boxed{\;W = K\cdot T_o,\qquad K=\dfrac{E}{1-E}\;}$$

> **含义**:等效率值(随处理器数 $p$ 增长所需的 $W$)**越小**,可扩展性**越好**。
> 例:$E=0.5\Rightarrow K=1$;$E$ 越接近 1,$K$ 越大,要求问题规模增长越快。
> 典型结论(π 积分 / 有限差分):等效率 $\Theta(p\log_2 p)$。

### 1.4 阶段并行模型

$$T_n=\sum_i\frac{T_1(i)}{\min(DOP_i,\,n)}+T_{\text{par}}+T_{\text{interact}},\qquad T_\infty=\sum_i\frac{T_1(i)}{DOP_i},\qquad N_{\max}=\max_i DOP_i$$
- $T_\infty$:**关键路径**(无限结点、不计开销的最短时间)。
- $N_{\max}$:**最大并行性**(使 $T_n=T_\infty$ 的最小结点数)。

### 1.5 Linpack / TOP500

- 浮点运算次数:$\;\dfrac{2}{3}N^3-2N^2$
- $R_{\max}=\dfrac{\frac{2}{3}N^3-2N^2}{\text{运行时间}}$
- $R_{\text{peak}}$=理论峰值,$R_{\max}$=实测持续最大,$N_{\max}$=达到 Rmax 时矩阵阶数。

### 1.6 通信模型

| 模型 | 点对点时间 |
|---|---|
| **Hockney** | $t_{\text{comm}}(m)=\alpha+m/\beta$($\alpha$=延迟,$\beta$=带宽) |
| 集合通信 | $T(m,n)=t_0(n)+m/r_\infty(n)$ |
| **LogP** | $L+2o$ |
| **BSP** superstep | $\max(\text{本地操作})+g\cdot h+l$ |

### 1.7 flop 计数规则

| 操作 | flop |
|---|---|
| 加 / 减 / 乘 | 1 |
| 除 / 开方 | 4 |
| sin / exp 等超越函数 | 8 |
| 比较、类型转换 | 1 |
| 下标计算、赋值 | 不单独计 |

## 三、必考概念对比表(选择/简答高频)

### 3.1 编程模型 vs 执行模型(L02)
- **编程模型(programming model)**:软件层面程序员看到的抽象(如共享内存、消息传递、数据并行)。
- **执行模型(execution model)**:硬件层面实际的执行方式(如 SIMD、MIMD、SIMT、多线程)。
- 二者**可以解耦**:同一编程模型可映射到不同执行模型。

### 3.2 Flynn 分类(L02)
| 类型 | 全称 | 含义 |
|---|---|---|
| **SISD** | Single Instruction Single Data | 传统串行单核 |
| **SIMD** | Single Instruction Multiple Data | 数据并行(向量机/GPU) |
| **MISD** | Multiple Instruction Single Data | 罕见(容错系统) |
| **MIMD** | Multiple Instruction Multiple Data | 多核/集群(最通用) |

### 3.3 SIMD vs SIMT(L04/L05)★必考
| | SIMD | SIMT |
|---|---|---|
| 全称 | Single Instruction Multiple Data | Single Instruction Multiple **Thread** |
| 代表 | 向量机、AVX/SSE intrinsics | **GPU(CUDA)** |
| 单位 | 向量寄存器 / 数据 lane | **warp(线程束,通常 32 线程)** |
| 控制 | 显式向量化、程序员管理 vector | 硬件自动分组 warp,**GPU 没有显式 vector** |
| 分支 | 掩码(mask)处理 | 分支发散(divergence),不同路径串行执行 |

### 3.4 Amdahl vs Gustafson / 强 vs 弱可扩展(L03)
| | Amdahl | Gustafson |
|---|---|---|
| 假设 | 问题规模**固定** | 问题规模**随系统扩大**(固定时间) |
| 对应 | **强可扩展性** | **弱可扩展性** |
| 加速比 | $n/(1+(n-1)\alpha)$,有上限 $1/\alpha$ | $n-(n-1)g$,近似线性 |
| 结论 | 串行瓶颈限制最大加速比 | 扩大问题规模可摆脱瓶颈 |

### 3.5 通信对比(L07/L10)
| | 双边通信(two-sided) | 单边通信(one-sided/RMA) |
|---|---|---|
| 操作 | Send / Recv **配对** | Put / Get / Accumulate |
| 同步 | 隐式匹配 | 显式 epoch(Fence / Lock-Unlock) |
| 内存 | 无需暴露 | 需 `MPI_Win` 暴露窗口 |

| | 阻塞(blocking) | 非阻塞(non-blocking) |
|---|---|---|
| 函数 | `MPI_Send/Recv` | `MPI_Isend/Irecv` |
| 完成检测 | 函数返回即完成 | 需 `Wait/Test` |
| 用途 | 简单 | 计算-通信重叠、避免死锁 |

## 五、高频考点与易错点 TOP 清单

1. **Amdahl 的 $\alpha$ 是串行比例**,问题规模固定;$1/\alpha$ 是加速比上限。别和 Gustafson 的 $g$ 混。
2. **等效率 $W=K\cdot T_o$,$K=\dfrac{E}{1-E}$**(不是 $\frac{1-E}{E}$);等效率越小可扩展性越好。
3. **加速比 vs 效率 vs 利用率**三个基准不同:串行时间 / n·并行时间 / 峰值速度。
4. **Linpack 浮点次数 $\frac{2}{3}N^3-2N^2$** 要记牢,$N$ 是矩阵阶数。
5. **SIMT vs SIMD** 区别:warp vs vector,GPU 用线程束而非显式向量。
6. **假共享**:根因是多线程写**同一缓存行**;解决靠 padding 让各线程数据落在不同缓存行。
7. **OpenMP 变量默认 shared**(除循环变量等),需 private 时要显式声明;循环迭代变量在 parallel for 中自动 private。
8. **MPI 死锁**:双方同时阻塞 Send(大消息缓冲满)/ 顺序错配 → 用非阻塞或 `MPI_Sendrecv` 解决。
9. **MPI 集合通信必须所有进程共同调用**(集体性),且参数要一致。
10. **自定义数据类型**必须 `MPI_Type_commit` 后才能用,用完 `MPI_Type_free`。
11. **网络直径**:拓扑中相距最远的两节点间最短路径的跳数;别和节点度混。
12. **flop 计数**:除/开方=4,sin/exp=8,不是 1。
13. **MPI_Comm_split**:按 color 分组、按 key 排序,生成新通信域(互不相交)。
14. **RMA 需要 epoch**:Put/Get 必须在 Fence 或 Lock-Unlock 纪元内才保证完成。
